---
name: layout_reorder_trap
description: Moving sections up the page is often proposed as a self-serve fix when the real problem is page length or label clarity
type: feedback
---

Reordering page sections to surface action content earlier should be challenged unless the page-length problem is addressed simultaneously.

**Why:** Moving TopIssues above detail tabs without shortening the page fragments user attention and may orphan adjacent sections (e.g., breakdown bars). The root cause is usually that the page is too long or section labels are too descriptive rather than imperative. A jump-link or copy change is almost always the smaller, more reversible fix.

**How to apply:** When a layout reorder is proposed for "discoverability," ask what the new page section order communicates holistically, and whether a copy change or anchor link solves the same problem at lower cost.
